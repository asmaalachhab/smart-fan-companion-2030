# Scrum — Smart Fan Companion 2030

Ce dossier ajoute **tous les livrables “Scrum”** (artefacts + rituels + règles de travail) pour piloter ton projet **Smart Fan Companion 2030**.

## Rôles
- **Product Owner (PO)** : porte la vision, priorise le backlog, valide les livraisons.
- **Scrum Master (SM)** : enlève les blocages, protège le temps de l’équipe, assure la qualité Scrum.
- **Dev Team** : dev front/back/db + QA + UX (1 seule équipe cross‑fonctionnelle).

> Si tu travailles solo : tu joues les 3 rôles, mais garde les rituels (même rapides) pour rester organisé.

## Cadence proposée
- **Sprint** : 2 semaines
- **Daily** : 10 minutes / jour
- **Sprint Planning** : 60–90 min
- **Review** : 45 min
- **Retro** : 30 min

## Board Kanban (dans GitHub Projects / Trello / Notion)
Colonnes recommandées :
1. **Backlog**
2. **Ready**
3. **In progress**
4. **In review (PR)**
5. **QA / Test**
6. **Done**

## Convention des tickets
- Préfixes : `EPIC`, `STORY`, `TASK`, `BUG`
- Estimation : Story Points (1,2,3,5,8,13)
- Chaque STORY doit avoir : *AC (acceptance criteria)* + *Definition of Done*.
