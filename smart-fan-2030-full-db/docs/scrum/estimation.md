# Estimation (Story Points)

On estime la **complexité** (pas les heures) avec Fibonacci : **1, 2, 3, 5, 8, 13**.

## Guide rapide
- **1** : petit changement UI / texte / petite route API
- **2** : formulaire simple + validation + test rapide
- **3** : feature moyenne (API + DB + UI) sans edge cases lourds
- **5** : feature complète avec plusieurs cas (ex : achat billet + stock)
- **8** : gros morceau (map + géoloc + filtres) → à découper
- **13** : trop gros → **découper**

## Comment estimer
1) Comparer à une story “référence” (ex : login = 3)
2) Planning Poker (PO + devs)
3) Décider un seul nombre

## Vitesse (velocity)
Après 1–2 sprints, calcule :
- points “Done” / sprint → ta capacité réelle.

## Capacité simple (si tu es solo)
- 2 semaines ≈ **8 à 12 points** (selon ton temps dispo)
